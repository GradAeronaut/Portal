# Инструкция: Как найти и закоммитить изменения

## Проблема
Изменения были внесены на сервере до настройки Git, и они не появились на Mac.

## Решение

### Шаг 1: Найти незакоммиченные изменения на сервере

```bash
cd /var/www/gradaeronaut.com

# Проверить статус
git status

# Показать все измененные файлы
git diff --name-only

# Показать новые файлы, не отслеживаемые Git
git ls-files --others --exclude-standard
```

### Шаг 2: Добавить и закоммитить изменения

```bash
# Добавить все изменения
git add -A

# Или добавить конкретные файлы
git add путь/к/файлу

# Закоммитить
git commit -m "Описание изменений"

# После коммита автоматически выполнится push в GitHub
```

### Шаг 3: Получить изменения на Mac

```bash
cd ~/Desktop/sinbad-portal-local  # или путь к вашей локальной копии

# Получить изменения с GitHub
git pull origin main
```

## Если изменения не видны в git status

Если файлы изменены, но не показываются в `git status`, возможно они:
1. В `.gitignore` - проверьте файл `.gitignore`
2. Не отслеживаются Git - добавьте их: `git add -f путь/к/файлу`

## Проверка конкретного файла

Если вы знаете, какой файл был изменен:

```bash
# Проверить, отслеживается ли файл
git ls-files --error-unmatch путь/к/файлу

# Посмотреть изменения в файле
git diff путь/к/файлу

# Добавить файл
git add путь/к/файлу
git commit -m "Обновлен файл"
```
